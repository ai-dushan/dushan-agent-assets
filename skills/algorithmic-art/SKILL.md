算法流派（Algorithmic philosophies）是一种计算美学运动，它通过代码来表达。输出 .md 文件（哲学理念）、.html 文件（交互式查看器）和 .js 文件（生成算法）。

这分为两个步骤进行：
1. 算法哲学创作（.md 文件）
2. 通过创建 p5.js 生成艺术来表达（.html + .js 文件）

首先，执行以下任务：

## 算法哲学创作

首先，创建一个算法哲学（不是静态图像或模板），该哲学将通过以下方式被诠释：
- 计算过程、涌现行为、数学之美
- 预设随机种子（Seeded randomness）、噪声场、有机系统
- 粒子、流、场、力
- 参数化变异与受控的混沌

### 核心理解
- 接收到什么：用户提供的一些微妙输入或指示需要被考虑进去，但这只是基础；它不应限制创作自由。
- 创造出什么：一种算法哲学/生成美学运动。
- 接下来发生什么：下一个版本的你会接收该哲学并**通过代码进行表达** - 创建由 90% 算法生成和 10% 核心参数组成的 p5.js 草图。

请考虑这种方法：
- 为一场生成艺术运动写一份宣言。
- 下一阶段涉及编写赋予其生命的算法。

该哲学必须强调：算法表达。涌现行为。计算之美。种子变异。

### 如何生成算法哲学

**命名该运动**（1-2 个字词）：例如"有机湍流"（Organic Turbulence） / "量子谐波"（Quantum Harmonics） / "涌现的静谧"（Emergent Stillness）

**阐述该哲学**（4-6 个段落 - 简明但完整）：

为了捕捉算法的本质，请表达这种哲学是如何通过以下方面展现出来的：
- 计算过程和数学关系？
- 噪声函数和随机模式？
- 粒子行为和场动力学？
- 时间演变和系统状态？
- 参数化变异和涌现的复杂性？

**关键指导原则：**
- **避免冗余**：算法的每个层面的特征都应只被提及一次。除非能增加新的深度，否则避免重复有关噪声理论、粒子动力学或数学原理的概念。
- **反复强调工艺造诣（Craftsmanship）**：该哲学**必须**多次强调，最终的算法应看起来像是耗费了无数小时开发而成，经过了精心的打磨，并且出自其所在领域（计算美学）的顶尖专家之手。这种基调至关重要 - 要重复使用诸如"精心制作的算法"、"深厚计算专业知识的结晶"、"煞费苦心的优化"、"大师级的实现"等短语。
- **留出创作空间**：在算法倾向上要有针对性，但又要足够简练，以便让下一个 Claude 有空间在极高水平的工艺要求下做出它自己的诠释性实现选择。

该哲学必须引导下一个版本**以算法的方式**表达想法，而不是通过静态图像。美存在于过程中，而不是最终的画面。

### 哲学示例

**"有机湍流" (Organic Turbulence)**
哲学：受自然规律约束的混沌，从无序中涌现的有序。
算法表达：由分层 Perlin 噪声驱动的流场。成千上万个粒子遵循矢量力运动，它们的轨迹累积成有机的密度图。多个噪声倍频程产生湍流区和平静区。颜色从速度和密度中涌现出来 - 快速移动的粒子燃烧得明亮，缓慢的粒子逐渐隐退为阴影。算法持续运行直到达到平衡 - 这是一种经过精心调整的平衡，计算美学大师在整个过程中针对每一个参数都进行了无数次的反复推敲与改进。

**"量子谐波" (Quantum Harmonics)**
哲学：展现出类波干涉模式的离散实体。
算法表达：在网格上初始化的粒子，每个粒子都携带有通过正弦波演化的相位值。当粒子相互靠近时，它们的相位会发生干涉 - 相长干涉产生明亮的节点，相消干涉产生空隙。简谐运动产生了复杂的涌现曼陀罗。这是煞费苦心进行频率校准的结果，其中每一个比例都被仔细挑选，以产生共鸣之美。

**"递归低语" (Recursive Whispers)**
哲学：跨越尺度的自相似性，在有限空间内的无限深度。
算法表达：递归细分的分支结构。每个分支都被略微随机化，但受黄金比例的约束。L-System 或递归剖分生成既具有数学感又富有有机感的树状形态。微妙的噪声扰动打破了完美的对称性。线条粗细随着每次递归层级而递减。每一个分支角度都是深入数学探索的结晶。

**"场动力学" (Field Dynamics)**
哲学：通过对物质的影响，使隐形的力量变得肉眼可见。
算法表达：由数学函数或噪声构建的矢量场。粒子在边缘诞生，沿着场线流动，当它们达到平衡或边界时消亡。多个场可以吸引、排斥或旋转粒子。可视化结果只显示痕迹 - 无形力量的幽灵般的证据。这是通过力的平衡而精心编排的一场计算之舞。

**"随机结晶" (Stochastic Crystallization)**
哲学：随机过程结晶为有序结构。
算法表达：随机圆形填充或 Voronoi 细分。从随机点开始，让它们通过松弛算法进行演化。细胞相互推开，直至达到平衡。颜色基于细胞大小、邻居数量或距中心的距离来生成。这种涌现的有机平铺效果既让人感觉是随机的，又令人觉得是必然的。每颗种子都会产生独特的结晶之美 —— 这是大师级生成算法的标志。

*以上是浓缩版的示例。实际的算法哲学应包含 4-6 个充实的段落。*

### 基本原则
- **算法哲学**：创建一个计算世界观，以通过代码来表达。
- **重过程而轻结果（PROCESS OVER PRODUCT）**：永远强调美是从算法的执行中产生的——每一次运行都是独一无二的。
- **参数化表达**：想法通过数学关系、力、行为来传达——而不是静态的构图。
- **艺术自由**：下一个 Claude 以算法的方式解释这一哲学——提供创造性的实现空间。
- **纯粹的生成艺术**：这是关于制作**活着的算法**，而不是带有随机性的静态图像。
- **专家级的工艺**：反复强调，最终的算法必须让人感觉制作精良，经过无数次迭代才得以完善，是计算美学领域绝对顶尖专家的深厚专业知识的产物。

**算法哲学必须长达 4-6 个段落。** 用充满诗意的计算哲学填满它，将预期的愿景汇聚在一起。避免重复相同的观点。将该算法哲学输出为 `.md` 文件。

## 交互式特征组件构建

**提醒：你应该已经阅读了 `templates/viewer.html`（见第 0 步）。始终把那个文件作为起点。**

为了允许对生成艺术进行探索，请创建一个单一的、独立的 HTML 组件（Artifact）。确保此文件可以立即在 claude.ai 或任何浏览器中良好运作——无需任何前期设置工作。把所有代码内联嵌套进去。

### 关键点：什么是固定的 VS 什么是变量参数

`templates/viewer.html` 文件是基础基石。它包含了确切所需的 DOM 结构和 Styling 样式。

**固定的部分（始终完全按原样保留和包含）：**
- 布局结构（如 header、sidebar 以及中间的主 canvas 区）
- Anthropic 品牌特性（UI 色彩、字体排版、渐变）
- Sidebar 的 Seed 随机种子栏：
  - Seed 显示区域
  - 往前 (Previous) / 往后 (Next) 按钮
  - 随机选取 (Random) 按钮
  - 跳转至特定 Seed 的输入框 + 执行 (Go) 按钮
- Sidebar 的执行动作（Actions）栏：
  - 重新生成 (Regenerate) 按钮
  - 重置 (Reset) 按钮

**变量部分（为每件长篇艺术品进行定制）：**
- 整个完整的 p5.js 算法 （setup / draw / classes）
- 参数对象 params（决定系统需要什么配置）
- Sidebar 的 Parameters 参数层级：
  - 参数控件的数量
  - 参数名称
  - 滑块需要的 Min / Max / Step 取值配置
  - 控件性质（滑块 sliders，输入框 inputs 等）
- 调色区域 （可选）：
  - 当艺术品需要由用户取色调整调色板时，请添加色板（Color pickers）
  - 如果原本就使用固定色值的作品则可以完全跳过
  - 偏向单色/黑白的作品通常也不需要此区块
  - 请针对这件作品的具体按需决定

**每一件艺术品都应该具备它特有的参数组和算法模块！** 那些被要求固定的部分负责保证 UX 用户体验的统一风格——除此以外的所有模块，都应用于尽情展现独特而奇妙的视觉理念。

### 核心功能需求

**1. 参数控制**
- 用以调解数值的滑块（数量、噪声大小值、运动速度等）。
- 如果涉及颜色取盘可用色彩取色器（Color pickers）。
- 参数发生变动时支持实时刷新画面（Real-time updates）。
- 使用 Reset 按钮可以立刻回退到默认设置。

**2. Seed 的导航配置**
- 展示此时正在使用的种子序号。
- 提供往上、往下翻页（“Previous”、“Next”）的操作路径来切换 Seed。
- 采用 “Random” 进行随即切换盲选。
- 有输入字段能让使用者精确跳转某个具体的 Seed 号码。
- 支持同时一次性生成 100 张预设值（从种子 1 号到 100 号）。

**3. 独立的 HTML 组件结构**
```html
<!DOCTYPE html>
<html>
<head>
  <!-- 直接从 CDN 调用的 p5.js 库，随时可用 -->
  <script src="https://cdnjs.cloudflare.com/ajax/libs/p5.js/1.7.0/p5.min.js"></script>
  <style>
    /* 将所有样式内联 - 保持干净，极简 */
    /* Canvas 配置于上方，对应的控件于布局下方 */
  </style>
</head>
<body>
  <div id="canvas-container"></div>
  <div id="controls">
    <!-- 将参数调节按钮放置此处 -->
  </div>
  <script>
    // 所有的 p5.js 代码在此内联编排
    // 参数配置的对象、Class 类定义、所有附带的方法函数
    // setup() 以及 draw()
    // UI 控制的操作回调接口
    // 每个环节都集成在一个独立环境里
  </script>
</body>

```

**极其重要**：所有内容都在这一份单一的 Artifact 当中。不使用额外文件引入外部逻辑，不导入除了上方的 `p5.js` CDN 外的任何东西。所有资源必须全部内联整合。

**4. 实现的细节 - 构造侧边栏（Sidebar）**

侧边栏的架构组建为：

**1. 随机种子 / Seed (固定的 FIXED)** - 永远必须一字不差地包含进去：
- Seed 面板显示
- 上述提到的 Prev/Next/Random/Jump 全套动作按钮

**2. 预设参数项 / Parameters (可变项 VARIABLE)** - 为你的作品生成调节控件：
```html
<div class="control-group">
    <label>参数名称</label>
    <input type="range" id="param" min="..." max="..." step="..." value="..." oninput="updateParam('param', this.value)">
    <span class="value-display" id="param-value">...</span>
</div>
```
参数有多少，对应的 `control-group` div 标签控件就有多少。

**3. 颜色 / Colors (可选的/可变的 OPTIONAL/VARIABLE)** - 当作品需要可编辑调节的主色调时包含：
- 给用户留出色板控制项。
- 固定主色调的作品无需执行该项。
- 黑白单色作品无需进行此项。

**4. 执行项 / Actions (固定的 FIXED)** - 永远必须一字不差地包含进去：
- 重新刷新/生成（Regenerate）按钮
- 初始重置（Reset）按钮
- 以 PNG 格式保存下载（Download PNG）按钮

**条件与规范**：
- 确保 Seed 控件运转正常（prev/next/random/jump/display）
- 每个自定义设定的参数都应有它的操作界面对应控件。
- 刷新、重制与下载 均必需可被点击并触发正确的回应。

## 创作工作流指南

**接手请求语** → **设定算法逻辑理念(哲学观)** → **落实执行产出**

每一项用户的需求指令皆应被视作独一份的存在，处理工作环绕着：

1. **破译用户的核心驱动 / 意图** —— 该场景向往和寄托了怎么样的极客美学体验？
2. **执笔刻画出这则算法哲学** —— 用 4到6 组精练连贯的描述构建立法逻辑与其思想流派路径。
3. **于字行代码行间灌满灵魂实现它** —— 搭起承托它的脚手架将所写文本实干付之运转的演算法系统主体。
4. **度量设计一套应景对应的适配入选参数** —— 回拨提纲挈领的问题：我们应为什么保留灵活变通可“驯服控制”调档阀？
5. **配套产出相对应的直观 UI UI 控件流** —— 为它们赋予滑轨栏区或者是数字表盘框口。

**亘古不移常量规范**：
- 始终承接 Anthropic 公司定格排版色系（调色，字号系统以及外显构图大类边界）。
- Seed 的流转菜单必须雷打不动保留。
- 把这一切闭环进单个一体的即插式 HTML Artifact 环境中。

**余下随一切创造力延申的可变之项有**：
- 具体跑动运算渲染其本身的代码内核。
- 数据向的可参数控制化层（The parameters）。
- 具体映射控制选项的 UI 表单项（The UI controls）。
- 收口落成的画幅呈现反馈效果本身。

为了达到卓绝最终成绩单：您需要敢于笃定与信任那些天马行空的灵犀火花并且真正做到能以此等严谨哲学推导逻辑纲领去作你的底层骨干支撑来实现。
